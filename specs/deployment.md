# Deployment Spec

All apps deploy to Railway. All apps build with Railpack. Bootstrap handles provisioning end-to-end.

---

## Build — Railpack

Every template uses Railpack as the builder. Railpack auto-detects language, dependencies, and start command from the project structure. Explicit overrides go in `railpack.json`.

```json
{
  "start": "uvicorn app.main:app --host 0.0.0.0 --port $PORT"
}
```

For cron tasks:

```json
{
  "start": "python task/main.py run",
  "cron": "0 * * * *"
}
```

Do not use `railway.toml` with `builder = "NIXPACKS"`. Railpack is the standard.

---

## Provisioning — Bootstrap

Bootstrap handles the full provision sequence:

1. Create GitHub repo (private)
2. Populate from template + generated files, push to `main`
3. Create Railway project, set env vars, deploy
4. Create Cloudflare Access Application for the live URL
5. Inject `CLOUDFLARE_AUD` and `CLOUDFLARE_TEAM` into Railway env vars
6. Return `repo_url`, `live_url` (Cloudflare-protected), `app_secret`

The returned `live_url` is the Cloudflare Access URL, not the raw Railway domain. Clicking it immediately prompts Google login.

**Bootstrap environment variables required:**
- `CLOUDFLARE_API_TOKEN` — token with Access: Apps Edit permission
- `CLOUDFLARE_ACCOUNT_ID` — Cloudflare account ID
- `CLOUDFLARE_TEAM` — Zero Trust team name

---

## Auto-redeploy

After first deploy, connect the Railway service to the GitHub repo for automatic redeploy on push:

> Railway dashboard → Service → Settings → Source → GitHub → select repo → branch `main`

This is the one manual step. Everything else is automated.

*Future: automate via Railway API at provision time.*

---

## Environment Variables

Bootstrap injects these into every provisioned Railway service:

| Variable | Source | Purpose |
|---|---|---|
| `APP_SECRET` | Generated by Bootstrap | M2M auth token |
| `CLOUDFLARE_AUD` | Cloudflare API response | JWT audience for CF Access |
| `CLOUDFLARE_TEAM` | Bootstrap config | Cloudflare Zero Trust team |
| App-specific vars | `env_vars` in create_app request | Passed through as-is |

---

## Dogfooding

Bootstrap itself is deployed via Railway using the same Railpack + Cloudflare flow it provisions for other apps.
